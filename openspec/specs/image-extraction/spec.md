## Requirements

### Requirement: Endpoint extracts Hebrew words and provides model-generated English translations
The system SHALL expose `POST /api/extract` accepting `{ image: string }` (base64-encoded, no data URI prefix) and returning `{ items: { hebrew: string, english: string }[] }`. The English translation for each item SHALL be generated by the model — not copied from any English or phonetic text visible in the notebook image.

#### Scenario: Successful extraction from a clear photo
- **WHEN** a valid base64 image containing Hebrew vocabulary is posted
- **THEN** the endpoint SHALL return HTTP 200 with `{ items }` where each item has a non-empty `hebrew` string (plain consonants, no niqqud) and a correct model-generated English translation

#### Scenario: Notebook contains phonetic transcription instead of English translation
- **WHEN** a valid base64 image is posted where the notebook has phonetic text (e.g. "yesh") next to a Hebrew word
- **THEN** the endpoint SHALL return a correct English translation (e.g. "there is / exists"), not the phonetic text

#### Scenario: Duplicate Hebrew word appears multiple times in image
- **WHEN** a valid base64 image is posted where the same Hebrew word appears more than once
- **THEN** the endpoint SHALL return that word only once in the `items` array

#### Scenario: No Hebrew vocabulary found in image
- **WHEN** a valid base64 image containing no Hebrew words is posted
- **THEN** the endpoint SHALL return HTTP 200 with `{ items: [] }`

### Requirement: Extraction errors return structured error responses
The system SHALL return structured JSON error responses (never unhandled exceptions) for all failure modes.

#### Scenario: OpenAI API error
- **WHEN** the OpenAI API returns an error (rate limit, network failure, etc.)
- **THEN** the endpoint SHALL return HTTP 502 with `{ error: string }` describing the failure

#### Scenario: Model returns malformed JSON
- **WHEN** GPT-4o returns output that cannot be parsed as `{ items: [...] }`
- **THEN** the endpoint SHALL return HTTP 422 with `{ error: "Failed to parse model output" }`

### Requirement: Route timeout is set to 30 seconds
The system SHALL export `maxDuration = 30` on the route to prevent Vercel's default 10s timeout from cutting off slow vision inference calls.

#### Scenario: Slow inference does not time out prematurely
- **WHEN** the GPT-4o vision call takes up to 30 seconds
- **THEN** the request SHALL remain open and return a valid response
